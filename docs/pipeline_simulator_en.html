Pipeline Logic Explanation

A. Preroll Synchronization Queue (Time Control)

CEF pushes frame data in a free-run loop, while DeckLink polls and pulls textures.
  • Preroll Wait: When the queue size grows from 0, consumption is paused for 3 cycles (approx. 100ms) to "bank" frames and absorb timing jitter.
  • Freeze on Underflow: When the queue runs empty, the last output frame is duplicated and the state machine resets to Preroll Wait.

B. Weave Interlace Compositing

if (y % 2 == 0) { pixel = SampleFiltered1(x, y); // Frame 1 } else { pixel = SampleFiltered2(x, y); // Frame 2 }
Even rows of vertical line coordinate y are mapped from the previous frame (F1), and odd rows from the latest frame (F2). In areas with motion, the mixing of consecutive frames within a single image causes interlacing artifacts ("comb-like tearing").

C. Vertical LPF (Anti-Flicker)

To suppress "line flicker" caused by extreme differences in colors between adjacent scan lines during interlaced output, a low-pass filter (vertical blur) is applied.
// 3-Tap Kernel: [0.25, 0.50, 0.25] pixel[y] = above*0.25 + center*0.50 + below*0.25; // 5-Tap Kernel: [0.0625, 0.25, 0.375, 0.25, 0.0625] pixel[y] = above2*0.0625 + above*0.25 + center*0.375 + ...

D. Unpremultiply (Alpha Restoration)

The CEF render buffer is in a Premultiplied Alpha state, where RGB values are pre-multiplied by alpha. To separate the transparency mask (Key) and the color signal (Fill) cleanly for DeckLink, we divide the RGB values by alpha to restore the original colors.
if (pixel.a > alphaThreshold) { pixel.rgb /= pixel.a; }
* If division is performed when the alpha value is too small (near transparent), division-by-zero or numerical instability occurs. Therefore, processing is skipped when it falls below the alphaThreshold.

E. Jitter & Noise Model (Jitter Simulation)

Mathematical modeling of "real-world latency and fluctuations" occurring between the asynchronous processes of CEF (producer) and DeckLink (consumer).
  • Gaussian Noise (σ): interval = base + Normal(0, σ)
    Microscopic fluctuations caused by OS thread schedulers and timer resolution limits. Typically, Windows timer resolution is around 1-2 ms, but under high load, fluctuations (σ) of 5-15 ms can occur.
  • Spike Delay (Burst): if (random() < Prob) interval += BurstMs
    Sudden thread suspension due to Garbage Collection (V8 GC) or heavy resource loading. The typical probability is 0.1% to 1%, but when it occurs, the pipeline pauses for 30-100 ms, quickly starving the buffers.
  • Clock Drift (PPM): interval *= (1.0 + PPM / 1,000,000)
    Physical frequency deviation between the PC motherboard clock and the DeckLink's internal crystal oscillator. For instance, at +50 PPM, the CEF-side clock runs slightly slower relatively, causing periodic buffer underflows (or overflows) every few tens of minutes or hours.
