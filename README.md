# CEFDecklink

This project integrates **Chromium Embedded Framework (CEF)** with **Blackmagic DeckLink** cards to output HTML/JS content as Key/Fill signals via SDI using DirectX 11.

## Features
- **CEF Integration**: Renders modern web content off-screen.
- **DirectX 11**: efficient texture sharing and processing.
- **DeckLink Output**: Supports External Keying (Fill on SDI Out, Key on SDI Out 2).
- **Format Support**: Automatically detects and locks to input signal (e.g., 59.94i).

## Prerequisites
- **Windows 10/11** (x64)
- **Visual Studio 2022** (Standard MSVC C++ toolchain)
- **CMake** (3.20 or later)
- **Blackmagic Desktop Video** drivers installed.

## Project Structure
- `src/`: Source code (C++, HLSL shaders)
- `vender/`: Third-party dependencies
  - `cef/`: CEF binary distribution (Release/Debug DLLs and resources)
  - `Blackmagic DeckLink SDK 15.3/`: DeckLink headers and libraries
- `build/`: Build artifacts (Generated by CMake)

## Build Instructions

1.  **Configure the project**:
    ```powershell
    cmake -S . -B build
    ```

2.  **Build (Debug)**:
    ```powershell
    cmake --build build --config Debug
    ```

## How to Run

**Important**: You must run the executable from the build directory where it was generated to ensure it can find the necessary CEF resource files (`icudtl.dat`, `*.pak`, etc.).

Run the following command from the project root:

```powershell
.\build\Debug\DeckLinkDX11.exe
```

### Controls
- **Ctrl + C**: Exit the application.
- **+ / -**: Adjust the Alpha Threshold (for keying).

## Troubleshooting
- **"Invalid file descriptor to ICU data"**: This happens if you run the executable from the wrong directory. Make sure to run it as shown above, or ensure all `*.pak` and `*.dat` files from `vender/cef/Resources` are next to the `.exe`.
- **Decklink not found**: Ensure Blackmagic Desktop Video drivers are installed and a compatible device is connected.
